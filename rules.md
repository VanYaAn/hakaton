# Правила работы с репозиторием

## Основные ветки
- 1. `main` - здесь всегда лежит код готовый к деплою. *СЮДА НАПРЯМУЮ КОММИТИТЬ НЕЛЬЗЯ!*
  2. `develop` - ветка текущей разработки. Сюда изменения попадают только через PR (Pull Request). *СЮДА НАПРЯМУЮ КОММИТИТЬ НЕЛЬЗЯ!*

## Процесс разработки

- 1. Создайте ветку для фичи:
```
# переходим на ветку develop
$ git checkout develop
# подтягиваем из удаленного репозитория изменения в ветке develop
$ git pull origin develop
# создаём ветку для фичи Формат: (bovinxx/feature/obsidian-export, limon4ik/feature/ai-parser)
$ got checkout -b username/feature/feature-name
```

- 2. Разрабатывайте и делайте частые (АТОМАРНЫЕ!) коммиты с понятными сообщениями (feat: добавил экспорт в Obsidian, fix: исправил парсинг дат)
 
- 3. Запуште изменения в удалённые репозиторий
```
$ git push origin username/feature/feature-name
```
Далее вы переходите на Github -> Pull Requests -> New Pull Request. Указываете в base ветку develop.

- 4. Дождитесь выполнения CI пайплайна, всё должно быть зелёным. Дождитесь ревью от остальных членов команды.
